# J.A.R.V.I.S. — голосовой ассистент на Python

Персональный голосовой ассистент для Windows в стиле Тони Старка: офлайн-распознавание речи, озвучка ответов, ИИ-диалог через Groq, футуристичный HUD-интерфейс на Tkinter и управление жестами через веб-камеру.

> Проект писался для себя и под свой компьютер, поэтому часть «личных» путей и ключей вынесена в конфиги и переменные окружения — см. раздел [Настройка](#-настройка) ниже.

---

## ✨ Возможности

- **Голосовое управление** — офлайн-распознавание речи через [Vosk](https://alphacephei.com/vosk/) (поддержка русского и казахского языков)
- **Озвучка ответов** — синтез речи через `edge-tts` с кэшированием готовых файлов
- **ИИ-диалог** — общение и динамические реплики через Groq API (модель `llama-3.1-8b-instant`)
- **Командный движок** — нечёткое сопоставление фраз (`difflib`), запуск приложений, открытие сайтов, таймеры, напоминания, заметки
- **Wikipedia** — ответы на вопросы «кто такой / что такое» с подгрузкой картинки
- **HUD-интерфейс** — анимированная приборная панель в стиле Iron Man (Tkinter + Pillow)
- **Управление жестами** — отдельный модуль на OpenCV + MediaPipe (`jarvis_graph.py`)
- **Каталог команд** — GUI-редактор пользовательских команд (`jarvis_settings.py`)

## 📁 Структура проекта

```
jarvis_python/
├── jarvis_hud.py              # Главный HUD-интерфейс (точка входа)

├── jarvis_ai.py                # Клиент Groq API (диалог и динамические реплики)
├── jarvis_tts.py                # Озвучка ответов через edge-tts

├── jarvis_config.example.json  # Шаблон конфига (скопировать в jarvis_config.json)
├── jarvis_state.example.json   # Шаблон файла состояния (язык интерфейса)
├── jarvis_memory.example.json  # Шаблон файла заметок
```

## 🛠 Требования

- **Windows** (используются `ctypes`/`winmm`, ярлыки `.lnk`, поэтому без серьёзной адаптации на Linux/macOS не запустится)
- Python 3.10+
- Модели Vosk для распознавания речи (скачиваются отдельно, см. ниже)
- Аккаунт [Groq](https://console.groq.com/) с бесплатным API-ключом (опционально — без ключа ассистент работает на заранее заданных фразах)

## 🚀 Установка

```bash
git clone https://github.com/<ваш-username>/<ваш-репозиторий>.git
cd <ваш-репозиторий>
pip install -r requirements.txt
```

### 1. Скачайте модели Vosk

Ассистент использует офлайн-распознавание речи. Скачайте модели с [alphacephei.com/vosk/models](https://alphacephei.com/vosk/models) и положите их, например, в `models/vosk-ru` и `models/vosk-kz` внутри проекта — `jarvis_data.py` ищет их там по умолчанию.

Если хотите хранить модели в другом месте, переопределите пути через переменные окружения:

```bash
set JARVIS_VOSK_MODEL_RU=D:\JARVIS\model
set JARVIS_VOSK_MODEL_KZ=D:\JARVIS\model-kz
```

### 2. Настройте конфиг

```bash
copy jarvis_config.example.json jarvis_config.json
```

Откройте `jarvis_config.json` и вставьте свой Groq API-ключ:

```json
{
  "groq_api_key": "ваш_ключ_здесь",
  "groq_model": "llama-3.1-8b-instant",
  "edge_tts_voice": "ru-RU-DmitryNeural",
  "tts_cache_dir": "tts_cache",
  "ai_enabled": true,
  "tts_enabled": true
}
```

`jarvis_config.json` добавлен в `.gitignore` и никогда не попадёт в репозиторий — можно не бояться случайно закоммитить ключ.

### 3. Создайте рабочие файлы состояния

```bash
copy jarvis_state.example.json jarvis_state.json
copy jarvis_memory.example.json jarvis_memory.json
copy custom_commands.example.json custom_commands.json
```

Эти файлы тоже в `.gitignore` — туда ассистент будет сохранять язык интерфейса, ваши заметки и пользовательские команды.

### 4. Запустите

```bash
python jarvis_hud.py
```

Либо запустите `run.bat` и выберите режим из меню: главный HUD, каталог команд или управление жестами.

## ⚙️ Настройка путей

В `jarvis_data.py` пути к ярлыкам приложений (`APPS`) по умолчанию строятся из рабочего стола текущего пользователя Windows (`%USERPROFILE%\Desktop`). Если каких-то ярлыков нет — соответствующая команда просто не сработает, это не сломает остальной функционал. Отредактируйте список `APPS`, чтобы добавить свои приложения.

Папка с собственными звуковыми файлами (`VOICE_DIR`) и пути к моделям Vosk переопределяются переменными окружения:

| Переменная | По умолчанию | Назначение |
|---|---|---|
| `JARVIS_VOICE_DIR` | `./voice` | Папка с дополнительными звуковыми файлами |
| `JARVIS_VOSK_MODEL_RU` | `./models/vosk-ru` | Модель распознавания русского языка |
| `JARVIS_VOSK_MODEL_KZ` | `./models/vosk-kz` | Модель распознавания казахского языка |
| `JARVIS_VOSK_MODEL_KZ_FALLBACK` | `./models/vosk-kz/vosk-model-small-kz-0.42` | Резервная модель для казахского |

## 🔒 Безопасность и приватность

- Реальный API-ключ Groq **не входит** в репозиторий — используется только `jarvis_config.example.json` с пустым полем
- Файлы с локальным состоянием (`jarvis_state.json`, `jarvis_memory.json`, `custom_commands.json`) и кэши (`tts_cache/`, `wiki_cache/`) исключены через `.gitignore`
- Пути к ярлыкам приложений строятся динамически от текущего пользователя, а не зашиты под конкретный компьютер

Перед тем как пушить свои изменения, проверяйте `git status` — конфиг с ключом и личные заметки не должны попадать в коммит.
